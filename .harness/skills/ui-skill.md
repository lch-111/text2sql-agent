# UI Skill：用 AI 搭建前端项目的标准化流程

## 1. 明确设计风格

- 选择风格方向：简洁、商务、科技风等。参考成熟产品的配色、字号、间距、阴影等视觉语言。
- 收集 2-3 个参考网站截图，让 AI 分析设计规律（提取颜色、圆角、布局特点），但不要直接复制代码。
- 输出设计风格说明：主色、辅色、背景色、文字颜色、边框圆角、按钮样式等。

## 2. 选择技术方案与组件库

- **技术栈**：React（推荐）或 Vue，配合 TypeScript 强类型约束。
- **组件库**：Ant Design、Element Plus、shadcn/ui 等成熟库，保证 UI 统一且减少手写组件。
- 让 AI 遵守：禁止重复造轮子，优先使用组件库提供的基础组件（Button、Modal、Table 等）。

## 3. 制定代码规范

- **目录结构**：`src/components`（通用组件）、`src/pages`（页面）、`src/hooks`（自定义 hooks）、`src/utils`（工具函数）、`src/styles`（全局样式）。
- **模块边界**：每个页面或功能模块独立，禁止跨模块直接引用内部实现。
- **复用规则**：同一 UI 模式出现 ≥2 次，必须封装为独立组件。

## 4. 设计样式系统（Design Token）

- 定义 CSS 变量或 JS 常量：`--primary-color`, `--border-radius`, `--font-size-base` 等。
- **禁止硬编码样式值**，所有颜色、间距、字号必须引用 Token。
- 支持暗色主题（可选）：提供 `light` 和 `dark` 两套 Token。

## 5. 编写实施计划（分阶段）

- **阶段一**：搭建项目骨架（路由、布局、组件库引入、样式系统）。
- **阶段二**：开发公共组件（Header、Sidebar、Card、Table 等）。
- **阶段三**：按页面或功能模块迭代（如登录页 → 工作台 → 列表页 → 详情页）。
- 每个阶段让 AI 生成可运行的代码，并确保前后端接口对接。

## 6. 与 AI 协作的注意事项

- 明确告诉 AI：遵循上述规范，使用组件库，不要生成冗余代码。
- 每次迭代只给一个小任务（如“实现用户头像下拉菜单”），完成后验收再继续。
- 定期让 AI 整理组件清单和样式变量，保持文档同步。

## 7. 验收标准

- 页面在不同分辨率下自适应（使用组件库的响应式布局）。
- 样式变更只需修改 Design Token，无需逐个调整组件。
- 无硬编码颜色值、无重复 UI 代码块。
